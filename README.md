  AI Phishing Detection Tool 

## Project Overview

This project aims to develop an AI-based phishing detection system that analyzes URLs and email content to identify potential cyber threats. Using machine learning techniques, the system evaluates suspicious patterns and provides a risk score to users, helping prevent phishing attacks and improving digital security.

## Objectives

Detect phishing websites using machine learning
Analyze URL-based features for threat detection
Provide a user-friendly interface for scanning links
Enhance awareness of cybersecurity risks


## Technologies Used

Python
Pandas & NumPy (Data Processing)
Scikit-learn (Machine Learning)
Flask (Backend API)
HTML, CSS, JavaScript (Frontend)


## Dataset

The project uses a phishing websites dataset containing features such as:

URL length
Presence of IP address
HTTPS usage
Number of subdomains
Domain age

## Features

- **ML-Powered Classification** — Scikit-learn model predicts Phishing vs Legitimate using 31 URL features
- **Dual-Signal Verdict** — Combines ML output + heuristic alerts for a smarter final decision (`Legitimate` / `Suspicious` / `Phishing`)
- **Domain Whitelist** — Major brands (Google, PayPal, GitHub etc.) are instantly cleared, eliminating false positives
- **Fuzzy Typosquat Detection** — RapidFuzz catches character-swap domains like `paypa1.com` or `gooogle.com` with 80% similarity threshold
- **Unicode Homograph Detection** — Flags IDN attacks using visually identical Unicode characters (e.g. Cyrillic `а` vs Latin `a`)
- **Shannon Entropy Scoring** — High-entropy domains commonly generated by phishing kits are flagged automatically
- **Phishing Keyword Scanning** — Word-boundary regex scans for `login`, `verify`, `bank`, `secure` etc. without false positives
- **IP Address Detection** — Flags raw IP-based URLs like `http://192.168.1.1/login`
- **Subdomain Depth Check** — Detects suspicious deep subdomain chains
- **HTTPS Validation** — Flags any URL not using a secure protocol
- **Risk Score (0–100)** — Quantified threat level beyond a simple binary label
- **REST API with CORS** — Clean JSON responses, ready to plug into any frontend
- **Graceful Fallback** — API runs even if the model file is missing, falling back to heuristics only
